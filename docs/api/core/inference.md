
Tagged-value entry point for factor-graph construction, called from LLVM codegen. Unpacks `num_vars` (must be a tagged INT64) and `var_dims_tensor` (a tensor `#(...)` or Scheme vector giving each variable's number of discrete states) and forwards to eshkol_make_factor_graph(). On success `result` is set to a HEAP_PTR tagged value wrapping the new graph; on a type mismatch, missing argument, or allocation failure `result` is set to the tagged NULL value.

**Parameters**

- `arena` — Arena used for all graph and belief allocations.
- `num_vars` — Tagged INT64: number of random variables.
- `var_dims_tensor` — Tagged tensor/vector of per-variable state counts.
- `result` — Destination tagged value (HEAP_PTR on success, NULL on failure).

### `eshkol_fg_add_factor_tagged`

```c
void eshkol_fg_add_factor_tagged(arena_t* arena,

 const eshkol_tagged_value_t* var_indices,

```

Tagged-value entry point for adding a factor to a graph, called from LLVM codegen. Unpacks `var_indices` (tensor or vector of variable indices) and `cpt_tensor` (tensor or vector of log-probabilities, with size equal to the product of the connected variables' dimensions) and builds/attaches a new factor to `fg` via eshkol_make_factor() and eshkol_fg_add_factor().

**Parameters**

- `arena` — Arena used for factor and CPT allocations.
- `fg` — Tagged HEAP_PTR wrapping the target eshkol_factor_graph_t.
- `var_indices` — Tagged tensor/vector of variable indices the factor connects.
- `cpt_tensor` — Tagged tensor/vector of log-probabilities for the CPT.
- `result` — Always set to the tagged NULL value.

> Note: `fg` is mutated in place; `result` is unconditionally set to the tagged NULL value regardless of success (the Scheme builtin discards the return value and relies on the in-place mutation).

```c
void eshkol_fg_infer_tagged(arena_t* arena,
 const eshkol_tagged_value_t* fg,
 const eshkol_tagged_value_t* max_iters,
 eshkol_tagged_value_t* result);
```

Tagged-value entry point for belief propagation, called from LLVM codegen. Runs eshkol_fg_infer() on `fg` with the given `max_iters` (defaults to 20 if not a tagged INT64) and a fixed convergence tolerance of 1e-6. On success, `result` is set to a HEAP_PTR wrapping a freshly allocated one-dimensional tensor containing every variable's belief vector, concatenated in variable order and converted from log-space back to probabilities. Set to the tagged NULL value if `fg` is not a valid factor-graph HEAP_PTR or allocation fails.

**Parameters**

- `arena` — Arena used for the result tensor and any BP scratch state.
- `fg` — Tagged HEAP_PTR wrapping the eshkol_factor_graph_t to run inference on.
- `max_iters` — Tagged INT64 iteration cap, or NULL/non-INT64 to use the default (20).
- `result` — Destination tagged value (HEAP_PTR tensor of beliefs on success, NULL on failure).
