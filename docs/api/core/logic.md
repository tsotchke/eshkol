
Tagged-value entry point for eshkol_unify(), called from LLVM codegen. Extracts a substitution pointer from `subst` (treated as the empty substitution if it is not a HEAP_PTR) and unifies `t1` with `t2` under it. On success `result` is set to a HEAP_PTR wrapping the extended substitution; on unification failure `result` is set to the tagged NULL value, which the Scheme surface displays as `#f`.

**Parameters**

- `arena` — Arena used for any new substitution allocated during unification.
- `t1` — First term to unify.
- `t2` — Second term to unify.
- `subst` — Tagged HEAP_PTR wrapping the substitution to unify under, or NULL/non-HEAP_PTR for empty.
- `result` — Destination tagged value (HEAP_PTR substitution on success, NULL on failure).

### `eshkol_display_substitution`

*Function* — line 348

```c
void eshkol_display_substitution(const eshkol_substitution_t* s, void* file);
```

Print a substitution as `{var -> term, var -> term, ...}`. Prints `{}` if `s` is NULL. Each bound term is formatted according to its tagged type (integers, doubles via eshkol_fprint_double(), booleans as `#t`/`#f`, nested logic variables, `()` for NULL, interned symbol strings for HEAP_PTR/HEAP_SUBTYPE_SYMBOL, or a generic `#<heap:N>` / `#<type:N>` placeholder for anything else).

**Parameters**

- `s` — Substitution to print, or NULL.
- `file` — Destination `FILE*`, or NULL to write to stdout.
