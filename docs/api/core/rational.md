Subtract two rationals: a - b. Computed via 128-bit intermediates to guard against int64 overflow before GCD-reduction.

**Parameters**

- `arena` — Arena to allocate the result from.
- `a` — Minuend, a rational pointer (eshkol_rational_t*) produced by eshkol_rational_create() or another rational operation.
- `b` — Subtrahend, likewise a rational pointer.

**Returns**

Newly allocated, GCD-reduced rational pointer, or NULL if the exact result overflows int64 (caller should fall back to double arithmetic).

Pointer-based binary dispatch for LLVM codegen. Equivalent to eshkol_rational_binary_tagged() but takes and writes tagged values by pointer, avoiding ABI issues with by-value eshkol_tagged_value_t struct passing/returning from generated code.

**Parameters**

- `a` — Left operand (tagged int, double, or rational).
- `b` — Right operand (tagged int, double, or rational).
- `op` — Operation selector: 0=add, 1=sub, 2=mul, 3=div.
- `result` — Tagged value written with the operation's result (rational or int if both operands are exact, double otherwise).
