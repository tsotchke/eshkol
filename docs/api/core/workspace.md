```c
 eshkol_tagged_value_t* result);
```

Tagged-value entry point for creating a workspace, called from LLVM codegen. Unpacks `dim` and `max_modules` from exact int64 tagged values and forwards to eshkol_make_workspace(). Follows the same alloca/store/call/load calling convention used for bignum intrinsics.

**Parameters**

- `arena` — Arena to allocate the workspace from.
- `dim` — Tagged int64 giving the workspace content dimension.
- `max_modules` — Tagged int64 giving the maximum number of modules.
- `result` — Output: HEAP_PTR tagged value wrapping the new workspace, or an ESHKOL_VALUE_NULL tagged value if any argument is invalid or allocation fails.

Tagged-value entry point for registering a module, called from LLVM codegen. Unpacks the workspace pointer from `ws,` extracts the module name from a string or symbol heap object in `name` (defaulting to "unnamed" if it cannot be resolved), and forwards to eshkol_ws_register().

**Parameters**

- `arena` — Arena used to copy the module name into workspace storage.
- `ws` — Tagged HEAP_PTR value wrapping the target workspace.
- `name` — Tagged value referencing a string or symbol heap object.
- `process_fn` — Tagged closure value: (tensor -> (cons double tensor)).
