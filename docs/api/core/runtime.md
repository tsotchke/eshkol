Push a new frame onto the current thread's Eshkol-level call stack. Codegen emits a call to this at every Eshkol function entry (paired with eshkol_frame_pop() at exit) so runtime errors can render an Eshkol-level traceback instead of a raw C unwind. `function_name` and `source_file` must be stable C strings (codegen emits them as globals); no copy is made.
