# v1.2-scale completion and production hardening plan

**Date**: 2026-04-17
**Scope**: close out v1.2-scale + full-codebase production hardening sufficient
for public (non-lab) deployment.
**Codename**: `v1.2-scale` — this is the codename for the entire v1.2 minor
version; every point release within v1.2.x carries this codename. Hardening
work ships under the same `scale` codename, not a separate one.
**Target tag**: `v1.2.1-scale` (or a later v1.2.x point release if the
hardening work spills across multiple ship windows).

This document plans (a) all remaining v1.2 feature work, (b) systematic
hardening of the whole substrate, (c) expanded edge-case test coverage, and
(d) stress/fuzz/security testing. It sits alongside `COMPILER_ROADMAP.md`
(which covers v1.3+) and supersedes `NOESIS_TRAJECTORY.md` for the v1.2
window only.

Hardening here means: assume an adversary can supply arbitrary source, input
data, network bytes, files, and JIT fragments. The compiler and runtime
must *either* reject them cleanly *or* handle them without memory-safety
violation, undefined behaviour, information leak, privilege escalation, or
silent data corruption.

## 3. Hardening streams — concrete plans

### 3.B Memory-safety audit

Targets (ordered by blast radius):

- `lib/core/arena_memory.cpp` — arena allocator. Integer-overflow-safe size
  computations, header-pointer arithmetic correctness, alignment guarantees.
- `lib/backend/llvm_codegen.cpp` (35 KLOC) — every `memcpy`, `CreateLoad`,
  `CreateGEP` with a computed offset. Particular focus on tagged-value
  packing / unpacking (where #131 bug lived).
- `lib/core/image_io.c`, `lib/core/model_io.cpp`, `lib/core/kb_persistence.cpp`
  — any read from untrusted files. Size fields in headers must bound the
  read.
- `lib/core/sexp_to_ast.cpp`, `lib/frontend/parser.cpp` — unbounded
  recursion in deeply-nested source; symbol interning table growth.
- `lib/agent/subprocess.esk` + `subprocess.h` — buffer sizes for captured
  stdout/stderr, env-var passthrough (no injection).
- `lib/agent/regex.esk` (PCRE2) — ReDoS prevention (catastrophic backtracking
  timeout).

Deliverables:
- Per-file issue list with file:line, severity, proposed fix.
- ASan build (`CMakeLists.txt` option `ESHKOL_ENABLE_ASAN=ON`) passing full
  test suite.
- UBSan build similarly.
- MSan build (harder — needs instrumented libc on Linux; best-effort).
- At least one regression test per issue fixed.

### 3.C Input-validation audit

For each ingest point: what bytes from where, what invariants must hold, is
each invariant checked before use?

| Ingest point | Source | Invariants |
|---|---|---|
| `read` / `open-input-string` | user code | UTF-8 validity, bounded length, balanced parens |
| parser | source file | max nesting depth, max symbol length, string-escape validity |
| `kb-load` | file on disk | magic bytes, version compatibility, tagged-value shape per slot |
| `model-load` (`.eshkol-model`) | file on disk | magic, tensor-dims bounded, shape product non-overflow |
| `image-read` (stb_image) | file on disk | dimensions × channels bounded against arena size |
| `json-parse` | user-supplied string | depth limit, number-literal bounds, UTF-8 |
| `csv-parse` | file on disk | row-count / column-count bounded, cell-length bounded |
| FFI extern | user code | pointer-type-compatibility |
| Python FFI (`eshkol.Context.eval`) | user code | same as parser |
| Subprocess env / argv | user code | no embedded NUL, bounded total size |
| HTTP response (qLLM / future server) | network | content-length sanity, header count cap, chunked-encoding correctness |
| SQLite bound params | user code | type match, size for bytevector blobs |

Deliverables:
- Per-ingest-point "contract" document.
- Fuzz harness per ingest point (stream K).
- Corpus of valid + invalid inputs for each.
- Explicit size-limit constants (`ESHKOL_MAX_DEPTH`, `ESHKOL_MAX_STR_LEN`,
  `ESHKOL_MAX_ARENA_ALLOC`) configurable at build and runtime.

### 3.D Concurrency hardening

Already addressed in #130 (worker-thread AD-tape + arena init). Remaining:

- Audit every `thread_local` variable in `lib/` for (a) zero-init correctness,
  (b) shutdown cleanup, (c) forked-process behaviour.
- Audit every use of `__current_ad_tape` and `__ad_mode_active` — currently
  non-TLS process globals per comment in arena_memory.cpp:40. Assess real
  race exposure and either (a) make them TLS (ABI work on ARM64 macOS), or
  (b) document "don't enable AD in parallel-map worker code" formally.
- TSan build (`ESHKOL_ENABLE_TSAN=ON`) passing a new
  `tests/concurrency_stress/` suite.
- Audit the work-stealing deque (`lib/backend/work_stealing_deque.h`,
  `epoch_manager.h`) for memory-ordering correctness under a formal
  happens-before model.
- Deadlock testing: parallel-map-in-parallel-map, parallel-map of
  parallel-fold, etc. (#125 is fixed but the broader family needs coverage.)

### 3.E Error-propagation audit

Find every place the runtime calls `eshkol_error(…)` followed by
`continue;` / `return nullptr;` / similar — those silently eat errors.
Convert to either:
- a typed exception raised to the caller (preferred), or
- an explicit return-value error that the caller must check (weaker, only
  for paths where exceptions aren't safe), or
- a `panic`-style abort with diagnostic (for invariant violations that
  indicate internal corruption).

Specific files known to have `eshkol_error` + fall-through today:
- `lib/core/arena_memory.cpp` — many allocation-failure paths return NULL
  silently.
- `lib/backend/*.cpp` — many codegen paths return nullptr after warning.

Deliverable: `ESHKOL_ERROR_MODE={raise|return|panic}` build option; default
to `raise` in v1.2.x-scale.

### 3.F Resource-management audit

Instrument the runtime to count every open fd, mmap region, arena block,
thread-pool worker, sqlite handle, and track every (alloc, free) pair.
Run the full test suite; report any imbalance (leak or double-free).

Tools: already have arena accounting in `arena_memory.cpp`. Add equivalent
for fd/mmap/sqlite via simple thread-local counters.

### 3.G Security surface

Priority vulnerability classes:

| Class | Example in Eshkol | Test plan |
|---|---|---|
| Path traversal | `(file-read "../../../etc/passwd")` | test canonicalisation in `lib/agent/*.esk` path handling |
| TOCTOU | `(file-exists? p)` → race → `(file-read p)` | use `openat` / single-syscall ops; test with threaded deleter |
| Symlink attack | temp file created via `mkstemp` but opened separately | audit all `mkstemp`/`mkdtemp` call sites |
| Command injection | `(subprocess-spawn (string-append "ls " user-input))` | document and enforce argv-list form (already done in subprocess.esk — verify) |
| SQL injection | `(sqlite-exec db (string-append "SELECT * FROM " x))` | enforce prepared-statement binding in agent/sqlite.esk |
| ReDoS | `(regex-match "(a+)+" str)` | pcre2 `PCRE2_MATCH_LIMIT_DEPTH` + timeout |
| Crafted binary | malformed `.kb`, `.eshkol-model`, `.png` | already have kb #117 fix; extend fuzzing |
| Integer overflow on allocation | `(make-tensor (list (* 1e18 1e18)))` | overflow check in every `arena_allocate_*_full` size calc |
| Supply chain | external `.so` via `(extern …)` | document that extern calls escape the sandbox |

Deliverable: `docs/SECURITY.md` threat model + CVE-reporting address +
supported-version matrix.

### 3.H Deterministic execution

- Remove or document every source of non-determinism: time, PID, uninit
  memory, threading interleaving, hash-table iteration order, PRNG without
  seed.
- Build with `-fno-delete-null-pointer-checks -ftrapv` where supported.
- Reproducible-build metadata (SOURCE_DATE_EPOCH) — binaries built from
  identical source must hash-compare equal.

### 3.I Edge-case test coverage (additional suites)

Existing: 179 tests, mostly happy-path + known-bug regressions. Add:

- `tests/edge_cases/arithmetic_boundaries.esk` — already present (34 tests);
  extend to 100+ (negative zero, subnormals, rational boundary between
  exact and inexact, bignum-float promotion, complex NaN).
- `tests/edge_cases/string_unicode.esk` — NEW. UTF-8 validity,
  normalisation (once #176 lands), empty string ops, max-length string,
  multi-byte char boundaries.
- `tests/edge_cases/tensor_shapes.esk` — NEW. Zero-length dims, 1-D / 2-D /
  N-D reshape invariants, broadcasting, non-contiguous strides, tensor of
  tensors.
- `tests/edge_cases/ad_gradients.esk` — NEW. Gradient through: pow with
  negative/zero base, sqrt at zero, log at zero, chained max/abs,
  discontinuous piecewise.
- `tests/edge_cases/kb_queries.esk` — NEW. Empty KB, 1-fact KB, huge KB
  (100K facts, time-boxed), pathological unification, circular logic-var
  bindings.
- `tests/edge_cases/parser_malformed.esk` — NEW. Unclosed parens, unknown
  reader macro, invalid UTF-8 source, overly-nested list, gigantic string
  literal.
- `tests/edge_cases/ffi_boundaries.esk` — NEW. Python FFI with each type
  shape; subprocess with non-UTF-8 output; SQLite with binary blobs.

Target: **1000+ total tests** before v1.2.x-scale ships.

### 3.J Stress testing

New suite `tests/stress/`:

- `tests/stress/24h_agent_loop.esk` — run a minimal agent (Selene-lite) for
  24h, check for memory growth, fd growth, thread count, panic.
- `tests/stress/parallel_at_scale.esk` — parallel-map of AD gradient on 1M
  items. Verify result correctness + timing bounds.
- `tests/stress/arena_pressure.esk` — allocate until arena grows to
  configured max, verify graceful rejection (no SEGV, exception raised).
- `tests/stress/fd_exhaustion.esk` — open 100K files in rapid succession,
  verify clean EMFILE handling.
- `tests/stress/large_kb_query.esk` — 1M-fact KB load + query. Verify
  latency bounds with hash-table path.
- `tests/stress/huge_tensor.esk` — 10GB tensor allocation. Verify OOM
  raises exception, doesn't crash.

All stress tests are time-boxed in CI (max 10 minutes) with longer
overnight runs in a separate workflow.

### 3.K Fuzzing

Per ingest point, a minimal fuzz driver:

- `fuzz/parser_fuzzer.c` (libfuzzer) — takes input bytes, calls
  `eshkol_parse_next_ast_from_string`, asserts no crash.
- `fuzz/read_fuzzer.c` — same for `read`.
- `fuzz/json_fuzzer.c`, `fuzz/base64_fuzzer.c` — stdlib parsers.
- `fuzz/kb_load_fuzzer.c`, `fuzz/model_load_fuzzer.c` — binary-format
  loaders.
- `fuzz/image_read_fuzzer.c` — stb_image (many pre-existing CVEs in upstream;
  pin to a known-patched revision).
- `fuzz/regex_fuzzer.c` — PCRE2 pattern + subject, time-limited.

Start with 100K iterations / driver in CI. Archive crash corpora in
`fuzz/corpus/`.

### 3.L Sanitizer builds

Add to `CMakeLists.txt`:

```cmake
option(ESHKOL_ENABLE_ASAN "Build with AddressSanitizer" OFF)
option(ESHKOL_ENABLE_UBSAN "Build with UndefinedBehaviorSanitizer" OFF)
option(ESHKOL_ENABLE_TSAN "Build with ThreadSanitizer" OFF)
option(ESHKOL_ENABLE_MSAN "Build with MemorySanitizer (Linux only, brittle)" OFF)
```

CI matrix expansion: 16 existing lanes + 4 sanitizer lanes. Sanitizer lanes
run the full test suite, all edge suites, and the stress suite (shorter
time-box).

## 5. Success criteria for v1.2.x-scale

Hard requirements:
- All 41 Noesis-audit items at M0 tier delivered.
- 1000+ tests passing (currently 179).
- ASan / UBSan / TSan full-suite runs clean.
- Fuzz harnesses run 24h without crash on any ingest point.
- Every C runtime function with an error path either raises a typed
  exception or returns an error the caller must check; no silent swallow.
- `docs/SECURITY.md` published with CVE reporting and supported-version
  policy.
- 16-lane CI matrix plus 4 sanitizer lanes all green.

Soft requirements:
- Memory footprint within documented bounds under stress.
- Long-running (24h) agent loop shows no leak (fd, thread, arena).
- Reproducible build: two builds of identical source hash-equal.
