## What Is Eshkol?

Eshkol is a compiled Scheme dialect targeting scientific computing, machine learning, and gradient-based optimization. It compiles `.esk` source to native binaries via LLVM, with zero garbage collection (arena-based memory), built-in automatic differentiation, and a novel consciousness engine for AI primitives.

## GPU Architecture

**3-tier dispatch**: XLA (>=100K elements) -> BLAS (>=64) -> SIMD (>=64) -> scalar

| Operation | Metal | CUDA |
|-----------|-------|------|
| Elementwise (+,-,*,/) | SF64 software float64 | Native f64 |
| Matrix multiply | SF64 + MPS | cuBLAS |
| Reduce (sum, mean) | SF64 | Native f64 |
| Softmax | SF64 | Native f64 |
| Transpose | SF64 | Native f64 |

Metal uses SF64 (software float64 via `uint2`) because Apple GPUs lack hardware double precision. All 21 GPU API functions implemented.

## Automatic Differentiation

Three modes, all production-ready:
- **Symbolic**: Compile-time AST transform (sum/product/quotient/chain rules)
- **Forward-mode**: Dual numbers, `derivative` operator
- **Reverse-mode**: Tape-based computational graphs, `gradient`/`jacobian`/`hessian`
- **Vector calculus**: divergence, curl, laplacian, directional-derivative
- 32-level nested tape stack

---

## Memory Model

- **Zero GC**: Arena allocation with bump-pointer (O(1) alloc)
- **OALR**: Ownership-Aware Lexical Regions
- **Tagged values**: 16 bytes = {type:8, flags:8, reserved:16, padding:32, data:64}
- **Tensors**: Homogeneous f64 as int64 bitpatterns, 64-byte aligned
- **Vectors**: Heterogeneous 16-byte tagged values
- **512MB stack** via linker flags, configurable via `ESHKOL_STACK_SIZE` env var

## Build System

```bash
# Lite build (system LLVM, GPU auto-detected, no XLA)
cmake -B build -DESHKOL_XLA_ENABLED=OFF && cmake --build build -j8

# XLA build (bundled LLVM/MLIR/StableHLO)
cmake -B build-xla -DESHKOL_XLA_ENABLED=ON -DSTABLEHLO_ROOT=deps/stablehlo && cmake --build build-xla -j8

# Compile stdlib
./build/eshkol-run lib/stdlib.esk --shared-lib -o build/stdlib.o

# Run all tests
./scripts/run_all_tests.sh

# Compile user program
./build/eshkol-run program.esk -o program && ./program
```

## Known Limitations

- **No GC**: Arena-based — long-running programs must manage scope lifetimes
- **Partial debugging**: DWARF line info via `-g`; breakpoints not yet implemented
- **Single-threaded codegen**: Compilation is single-threaded (runtime is parallel)
- **Escape analysis**: Region-based with conservative heap fallback

## Architecture Overview

```
Source (.esk)
    │
    ▼
Parser (recursive descent) ──► AST
    │
    ▼
Type Checker (bidirectional, HoTT) ──► Typed AST
    │
    ▼
LLVM Codegen (21 specialized modules)
    │
    ├──► LLVM IR ──► Native Object ──► Linked Binary
    │
    └──► XLA Path (optional)
         ├──► StableHLO Ops ──► MLIR ──► HW-optimized kernel
         └──► LLVM-direct fallback
```

**Codegen modules**: ArithmeticCodegen, CollectionCodegen, TensorCodegen, StringIOCodegen, BindingCodegen, TaggedValueCodegen, SystemCodegen, FunctionCache, TailCallCodegen, ParallelCodegen, XLACodegen, StableHLOEmitter, XLATypes, GPUMemory, BLASBackend
